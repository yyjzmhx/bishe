## 🎵 项目简介

MusicAI 是一个基于 AI 技术的个性化音乐推荐系统，支持音频文件上传、AI 智能分析、音乐推荐、评论互动等功能。系统采用前后端分离架构，后端使用 Spring Boot，前端使用 Vue 3。

### 核心功能

- 🎤 **音频上传与分析**：支持多种音频格式上传，AI 智能分析音频特征
- 🎯 **智能推荐**：基于音频分析结果，推荐相似风格的音乐
- 💬 **评论互动**：支持评论、回复、点赞等社交功能
- 📊 **用户管理**：完整的用户注册、登录、个人信息管理
- 🔔 **消息通知**：系统通知、消息推送功能
- 🎨 **主题切换**：支持亮色/暗色主题切换

## 🐳 Docker Compose 一键部署

### 1. 启动 Docker 服务

使用 Docker Compose 一键启动所有必需服务（推荐方式）：

```bash
# 进入项目根目录
cd PersonalMusicSystem

# 启动所有服务（MySQL、Redis、MinIO）
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 2. 验证服务启动

#### MySQL
```bash
# 检查 MySQL 是否运行
docker ps | grep mysql

# 连接 MySQL（可选）
docker exec -it music-mysql mysql -uroot -proot
```

#### Redis
```bash
# 检查 Redis 是否运行
docker ps | grep redis

# 连接 Redis（可选）
docker exec -it music-redis redis-cli
```

#### MinIO
```bash
# 检查 MinIO 是否运行
docker ps | grep minio

# 访问 MinIO 控制台
# 浏览器打开: http://localhost:9001
# 用户名: admin
# 密码: 12345678
```

### 3. 初始化 MinIO 存储桶

1. 访问 MinIO 控制台：http://localhost:9001
2. 使用账号密码登录（admin / 12345678）
3. 点击左侧菜单 "Buckets"
4. 点击 "Create Bucket"
5. 输入存储桶名称：`music-storage`
6. 点击 "Create Bucket" 完成创建

### 4. 停止和清理服务

```bash
# 停止所有服务
docker-compose down

# 停止并删除数据卷（⚠️ 会删除所有数据）
docker-compose down -v

# 重启服务
docker-compose restart
```

## 🐳 Docker 单独部署（不使用 Docker Compose）

如果不想使用 Docker Compose，可以单独使用 Docker 命令启动各个服务。

### Redis 部署

#### 1. 拉取 Redis 镜像
```bash
docker pull redis:7-alpine
```

#### 2. 创建并启动 Redis 容器
```bash
docker run -d \
  --name music-redis \
  -p 6379:6379 \
  -v redis_data:/data \
  --restart always \
  redis:7-alpine \
  redis-server --appendonly yes
```

#### 3. 验证 Redis 运行
```bash
# 检查容器状态
docker ps | grep redis

# 测试 Redis 连接
docker exec -it music-redis redis-cli ping
# 应返回: PONG
```

#### 4. 常用 Redis 命令

```bash
# 进入 Redis 容器
docker exec -it music-redis sh

docker exec -it music-redis redis-cli

docker start music-redis
```

## ⚙️ 配置说明

### 1. 后端配置 (`src/main/resources/application.yml`)

#### 数据库配置
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/music_system?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root          # 修改为您的 MySQL 用户名
    password: root          # 修改为您的 MySQL 密码
```

#### Redis 配置（验证码存储）

**重要**: Redis 用于存储邮箱验证码，必须配置正确

```yaml
spring:
  data:
    redis:
      host: localhost       # Redis 主机地址（Docker: localhost）
      port: 6379           # Redis 端口
      password:            # Redis 密码（Docker 默认无密码）
      database: 0          # Redis 数据库编号
      timeout: 5000         # 连接超时时间（毫秒）
```

**验证码存储说明**:
- 验证码存储在 Redis 中，Key 格式：`verification_code:{type}:{email}`
- 注册验证码过期时间：10分钟
- 密码重置验证码过期时间：1分钟
- 验证成功后自动删除（防止重复使用）

#### MinIO 配置
```yaml
minio:
  endpoint: http://localhost:9000        # MinIO API 地址（建议使用 ipconfig 获取的本机 IP 地址，如 http://192.168.1.100:9000）
  access-key: admin                      # MinIO 访问密钥
  secret-key: 12345678                  # MinIO 秘密密钥
  bucket-name: music-storage             # 存储桶名称
  base-url: http://localhost:9000/music-storage  # 基础 URL（建议使用本机 IP 地址）
```

**获取本机 IP 地址**:
- **Windows**: 打开命令提示符，输入 `ipconfig`，查找 "IPv4 地址"
- **Linux/Mac**: 打开终端，输入 `ifconfig` 或 `ip addr`，查找 inet 地址

**示例配置**（假设本机 IP 为 192.168.1.100）:
```yaml
minio:
  endpoint: http://192.168.1.100:9000
  base-url: http://192.168.1.100:9000/music-storage
```

#### AI 服务配置（阿里云通义千问）

**重要**: 需要申请阿里云 DashScope API Key

1. 访问 [阿里云 DashScope 控制台](https://dashscope.console.aliyun.com/)
2. 注册/登录账号
3. 创建 API Key
4. 配置到 `application.yml`:

```yaml
ai:
  dashscope:
    api-key: sk-your-api-key-here        # ⚠️ 替换为您的 API Key
    base-url: https://dashscope.aliyuncs.com/api/v1
    chat:
      options:
        model: qwen-turbo                # 模型名称
        temperature: 0.7                 # 温度参数
```

#### 邮件服务配置（QQ邮箱SMTP）

**重要**: 需要开启 QQ 邮箱的 SMTP 服务并获取授权码

1. 登录 QQ 邮箱
2. 设置 → 账户 → 开启 SMTP 服务
3. 生成授权码（不是邮箱密码）
4. 配置到 `application.yml`:

```yaml
spring:
  mail:
    host: smtp.qq.com
    port: 587
    username: your-email@qq.com          # ⚠️ 替换为您的 QQ 邮箱
    password: your-authorization-code    # ⚠️ 替换为您的授权码（不是密码）
```

#### JWT 配置
```yaml
jwt:
  secret: PersonalMusicSystemSecretKey2024ForJWTTokenGenerationMustBeLongEnoughForSecurity
  expiration: 86400000                   # Token 过期时间（毫秒），默认24小时
```

### 2. 前端配置

前端配置主要在环境变量文件中：

#### 开发环境 (`MusicVue/.env.development`)
```env
VITE_API_BASE_URL=http://localhost:8080/api
```

#### 生产环境 (`MusicVue/.env.production`)
```env
VITE_API_BASE_URL=http://your-domain.com/api
```

## 🚀 功能操作流程

### 1. 用户注册流程

```
用户访问注册页面
    ↓
输入用户名、密码、邮箱等信息
    ↓
点击"发送验证码"按钮
    ↓
系统生成6位数字验证码
    ↓
验证码存储到 Redis（10分钟过期）
    ↓
发送验证码邮件到用户邮箱（QQ邮箱SMTP）
    ↓
用户查收邮件，输入验证码
    ↓
系统验证验证码（从 Redis 读取并校验）
    ↓
验证成功，删除验证码（防止重复使用）
    ↓
创建用户账号（密码使用 BCrypt 加密）
    ↓
注册成功，跳转到登录页面
```

### 2. 音频上传与分析流程

```
用户登录系统
    ↓
进入"音频分析推荐"页面
    ↓
选择音频文件上传（支持 mp3, wav, flac 等格式）
    ↓
文件上传到 MinIO 对象存储
    ↓
创建上传记录（状态：上传中）
    ↓
调用阿里云 DashScope AI 服务分析音频
    ↓
AI 分析音频特征（风格、情绪、乐器等）
    ↓
更新上传记录（状态：分析完成）
    ↓
根据分析结果，从数据库查询相似音乐
    ↓
生成推荐列表
    ↓
用户查看分析报告和推荐结果
```

### 3. 评论互动流程

```
用户浏览音乐详情页
    ↓
查看评论列表（支持最新/最热排序）
    ↓
点击"回复"按钮
    ↓
展开回复输入框
    ↓
输入回复内容
    ↓
提交回复
    ↓
后端验证用户身份（JWT Token）
    ↓
保存回复到数据库（关联父评论ID）
    ↓
更新父评论的回复数量
    ↓
刷新评论列表，显示新回复
```

### 4. 播放列表流程

```
用户浏览音乐列表
    ↓
点击"添加到播放列表"按钮
    ↓
音乐添加到 Pinia Store 的播放列表
    ↓
点击播放按钮
    ↓
从 MinIO 获取音频文件 URL
    ↓
使用 HTML5 Audio API 播放音频
    ↓
实时更新播放进度
    ↓
支持播放模式切换（顺序/单曲循环/随机）
    ↓
支持上一首/下一首切换
```

## 🔍 常见问题

### 1. Docker 服务启动失败

**问题**: `docker-compose up -d` 失败

**解决方案**:
```bash
# 检查端口是否被占用
netstat -ano | findstr :3306
netstat -ano | findstr :6379
netstat -ano | findstr :9000

# 停止占用端口的进程，或修改 docker-compose.yml 中的端口映射
```

### 2. MySQL 连接失败

**问题**: `Communications link failure`

**解决方案**:
- 检查 MySQL 服务是否启动
- 检查 `application.yml` 中的数据库配置是否正确
- 检查防火墙是否开放 3306 端口
- 确认数据库用户权限

### 3. Redis 连接失败

**问题**: `Unable to connect to Redis`

**解决方案**:
- 检查 Redis 服务是否启动: `docker ps | grep redis`
- 检查 `application.yml` 中的 Redis 配置
- 如果使用 Docker，确保容器正常运行

### 4. MinIO 连接失败

**问题**: `The Access Key Id you provided does not exist`

**解决方案**:
- 检查 MinIO 服务是否启动: `docker ps | grep minio`
- 确认 `application.yml` 中的 Access Key 和 Secret Key 与 MinIO 配置一致
- 确认存储桶 `music-storage` 已创建

### 5. 验证码发送失败

**问题**: 邮箱验证码无法发送

**解决方案**:
- 检查 QQ 邮箱 SMTP 服务是否开启
- 确认使用的是授权码（不是邮箱密码）
- 检查 `application.yml` 中的邮件配置
- 查看后端日志确认错误信息

### 6. AI 分析失败

**问题**: `AI service error` 或 `API key invalid`

**解决方案**:
- 确认已申请阿里云 DashScope API Key
- 检查 `application.yml` 中的 API Key 配置
- 确认账户有足够的余额
- 检查网络连接是否正常

### 7. 前端无法连接后端

**问题**: `Network Error` 或 `CORS error`

**解决方案**:
- 确认后端服务已启动（http://localhost:8080）
- 检查前端 `.env.development` 中的 API 地址配置
- 确认后端 `WebConfig.java` 中已配置跨域

### 8. 验证码验证失败

**问题**: `验证码错误或已过期`

**解决方案**:
- 确认验证码在有效期内（注册验证码10分钟，密码重置1分钟）
- 检查 Redis 服务是否正常运行
- 确认验证码格式正确（6位数字）
- 查看后端日志确认 Redis 中的验证码

```bash
# 1. 克隆项目（如果还没有）
```
